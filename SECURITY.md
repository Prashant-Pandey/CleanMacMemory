# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

**Do not** publicly disclose security vulnerabilities. Instead, please email security@example.com with:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

We will:
- Acknowledge receipt within 48 hours
- Investigate the issue
- Provide a timeline for a fix
- Credit you in the release notes (if desired)

## Security Best Practices

This application:
- Contains no external dependencies (reducing attack surface)
- Requests minimal macOS permissions
- Performs no network communication
- Stores no persistent user data
- Is fully open-source and auditable

## Code Signing

CleanMacMemory is signed with an Apple Developer ID to ensure integrity and authenticity.

## Updates

Keep your copy of CleanMacMemory updated to receive security patches and improvements.
